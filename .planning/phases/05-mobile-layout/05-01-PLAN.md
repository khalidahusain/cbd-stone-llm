Create the mobile-specific dashboard wrapper at `frontend/src/components/MobileDashboard.tsx`.

This component is only rendered on mobile viewports (visibility controlled by App.tsx in Plan 02 via `md:hidden`). It reads prediction from ChatContext and renders a collapsible dashboard panel above the chat.

Implementation:

1. **Import dependencies:**
   - `useChatState` from context
   - `ProbabilityDisplay`, `GuidanceBar`, `CostTable`, `InterpretationGuide`, `Abbreviations`, `CholangitisOverlay` from dashboard subcomponents
   - `CollapsibleSection` from `./CollapsibleSection`
   - `PredictionResult` type from `../types/api`

2. **Risk tier background color map** (per D-07 — visually distinct header colored by risk tier):
   ```typescript
   const TIER_BG_COLORS: Record<string, string> = {
     low: "bg-green-50 border-green-200",
     intermediate: "bg-amber-50 border-amber-200",
     high: "bg-orange-50 border-orange-200",
     very_high: "bg-red-50 border-red-200",
   };
   const TIER_TEXT_COLORS: Record<string, string> = {
     low: "text-green-700",
     intermediate: "text-amber-700",
     high: "text-orange-700",
     very_high: "text-red-700",
   };
   const TIER_LABELS: Record<string, string> = {
     low: "Low Risk",
     intermediate: "Intermediate Risk",
     high: "High Risk",
     very_high: "Very High Risk",
   };
   ```

3. **Component logic:**
   - Read `prediction` from `useChatState()`.
   - If `prediction` is null, return `null` (per D-03 — completely hidden until first prediction).
   - Use `useState(true)` for the main dashboard expand/collapse (`dashboardExpanded`). Default to expanded on first appearance so the clinician sees the result.
   - The entire component wrapper should have a subtle entrance animation: `animate-in` effect. Use Tailwind classes: wrap in a div with `transition-all duration-300` and conditionally apply `opacity-0 translate-y-[-8px]` initially then remove after mount using a useEffect + setTimeout(0) pattern with a `mounted` state boolean. This gives a smooth slide-down appearance when the first prediction arrives.

4. **Collapsed header (per D-05):**
   - A `button` with `w-full` that toggles `dashboardExpanded`.
   - Background colored by risk tier (TIER_BG_COLORS).
   - Layout: `flex items-center justify-between px-3 py-2 rounded-lg border`.
   - Left side: `{prediction.probability}% — {TIER_LABELS[prediction.risk_tier]}` in TIER_TEXT_COLORS, `text-sm font-bold`.
   - Right side: chevron SVG (same as CollapsibleSection) that rotates when expanded. `w-4 h-4` in TIER_TEXT_COLORS.
   - Add `text-[10px] text-gray-500` label below the probability text: "Tap to expand/collapse".

5. **Expanded content (per D-08):**
   - Wrapped in a `div` with `overflow-hidden transition-all duration-200`. Use `max-h-0 opacity-0` when collapsed, `max-h-[2000px] opacity-100` when expanded (large max-h since content is variable).
   - Inside, a `div` with `p-3 space-y-3 bg-white border border-gray-200 rounded-b-lg`:
     - `<ProbabilityDisplay prediction={prediction} />` — always visible (D-08)
     - `<div className="relative">` containing `<GuidanceBar probability={prediction.probability} />` and conditional `<CholangitisOverlay>` — always visible (D-08)
     - Recommended intervention text block — always visible (D-08):
       ```
       <div className="text-center">
         <p className="text-xs text-gray-500">Suggested next step in management</p>
         <p className="text-sm font-bold text-gray-800">{prediction.recommended_intervention}</p>
       </div>
       ```
     - `<CollapsibleSection title="Cost-Weighted Interventions">` wrapping `<CostTable costs={prediction.cost_estimates} />` (per D-09, D-10)
     - `<CollapsibleSection title="Interpretation Guide">` wrapping `<InterpretationGuide />` (per D-09, D-10)
     - `<CollapsibleSection title="Abbreviations">` wrapping `<Abbreviations />` (per D-09, D-10)

6. **Export:** `export default function MobileDashboard()` — no props needed, reads from context.
